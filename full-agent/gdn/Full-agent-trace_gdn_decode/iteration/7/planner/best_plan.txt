2. **Optimal Performance Retention**: It preserves the zero-barrier, fully unrolled FMA logic that achieved the 827x peak score for $K=128$.
3. **Robust Generalization**: Expanding templates to $K \in \{64, 128, 256\}$ acts as a deterministic "multi-start" equivalent, allowing the compiler to generate multiple highly optimized, register-locked solutions for different dimension spaces.

2. **Phase 2: The Optimization Loop (Template Metaprogramming Generation)**
    * **Algorithm**: Instead of a Python runtime search, we use C++ Template Metaprogramming as the "solver". The compiler acts as the optimizer, generating distinct, optimal machine code for a defined set of discrete states.
    * **Action**: Define the `gdn_decode_kernel_fast` as a template over `int K` and `int V_UNROLL`.
    * **Loop (Multi-State Instantiation)**: Instruct the compiler to explicitly instantiate this template for the set of standard target dimensions: $K \in \{64, 128, 256\}$.
    * **Optimization Logic**: Inside the templated kernel, use `#pragma unroll` for all $K$-dependent reductions. Because $K$ is a template parameter, the compiler will unroll the loop exactly $K/4$ times, mathematically guaranteeing maximum FMA pipelining without branch divergence.

3. **Phase 3: The "Safety Valve" (CRITICAL Runtime Deterministic Dispatch)**

This generic kernel uses dynamic loops (`for (int k = tx; k < K; k += blockDim.x)`) and scalar memory access, guaranteeing that even if a strange dimension like $K=117$ is passed, the exact mathematical result is computed safely, ensuring a score $\ge 1.0$.
