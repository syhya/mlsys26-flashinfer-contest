*   **Variables ($X$)**: The data types and intrinsics used in the inner loop (e.g., `__nv_bfloat162` vs `float`).

*   If the BF16 math violates the correctness constraint, the executor must revert to FP32 for accumulation. However, loading/storing can definitely be vectorized using `float4` or `int4` equivalents.

**Why this solution?:**
