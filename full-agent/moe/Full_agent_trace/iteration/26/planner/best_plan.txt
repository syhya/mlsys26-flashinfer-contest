### Phase 1: Mathematical Definition & Scale Analysis

**Step 1.1: Understand the exact block-scale semantics for GEMM2**

The operation to implement is:
```
O[j, h] = routing_weight[j] * sum_{i=0}^{I-1}( C[j,i] * W2[e,h,i] * W2_scale[e, h//128, i//128] )
```
where:
- `C[j,i] = silu(G1[j, I+i]) * G1[j, i]` (SwiGLU output, float32)
- `W2[e, h, i]` is FP8, shape `[32, 7168, 2048]`
- `W2_scale[e, h//128, i//128]` is float32, shape `[32, 56, 16]`

When tiling with BLOCK_N=128 (over H) and BLOCK_K=128 (over I), each tile covers exactly ONE (h_block, i_block) pair, so W2_scale for a given tile is a SCALAR.

**Step 1.2: Define the FP8 tensor core path for GEMM2**

To use FP8 tensor cores, BOTH A and B must be FP8. Since C (SwiGLU output) is float32 with arbitrary magnitude, we must quantize it per-tile:
```
c_tile[m,k] ∈ R^{BLOCK_M × BLOCK_K}   (float32, SwiGLU output tile)
c_scale = max(|c_tile|) / 448.0        (scalar per tile — NOT per row)
c_fp8 = clamp(c_tile / c_scale, -448, 448).to(fp8_e4m3fn)
```

Then the FP8 dot with exact dequantization:
```
dot_result[m,n] = tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32)  # FP8 TCs
acc[m,n] += dot_result[m,n] * c_scale * w2_scale[n_block, k_block]  # exact
```

This is mathematically exact because:
```
sum_k(c[m,k] * W2[k,n]) * W2_scale
= sum_k(c_fp8[m,k] * c_scale * W2[k,n]) * W2_scale
= c_scale * dot(c_fp8, W2)[m,n] * W2_scale
```

The c_scale is a SCALAR per tile so it broadcasts correctly over the [BLOCK_M, BLOCK_N] output tile.

**Step 1.3: Verify GEMM1 is kept unchanged**

The parent's GEMM1 kernel is proven correct (A-scale indexing was fixed in iteration 46817723). Keep it verbatim. The only modification in this plan is GEMM2.

### Phase 3: New GEMM2 Kernel with FP8 Tensor Cores

**Step 3.1: Replace GEMM2 kernel with FP8 TC version**

The critical change: instead of `w2_f32 = w2_fp8.to(tl.float32) * w2_scale; tl.dot(swiglu, w2_f32)`, we do:

```python
@triton.autotune(
    configs=[
        triton.Config({"BLOCK_M": 16, "BLOCK_N": 128}, num_warps=4, num_stages=4),
        triton.Config({"BLOCK_M": 32, "BLOCK_N": 128}, num_warps=4, num_stages=5),
        triton.Config({"BLOCK_M": 64, "BLOCK_N": 128}, num_warps=4, num_stages=4),
        triton.Config({"BLOCK_M": 64, "BLOCK_N": 128}, num_warps=8, num_stages=3),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=8, num_stages=3),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=8, num_stages=4),
    ],
    key=["MAX_TK"],
)
@triton.jit
def grouped_gemm2_fp8_kernel(
    G1_ptr, W2_ptr, W2_scale_ptr, Routing_weights_ptr, Out_ptr,
    expert_offsets_ptr,
    I: tl.constexpr, H: tl.constexpr, MAX_TK,
    stride_g1m, stride_g1k,
    stride_w2e, stride_w2n, stride_w2k,
    stride_w2scale_e, stride_w2scale_n, stride_w2scale_k,
    stride_outm, stride_outn,
    BLOCK_K: tl.constexpr, BLOCK_M: tl.constexpr, BLOCK_N: tl.constexpr,
):
    pid_n = tl.program_id(0)
    pid_m = tl.program_id(1)
    pid_e = tl.program_id(2)

    start_idx = tl.load(expert_offsets_ptr + pid_e)
    end_idx = tl.load(expert_offsets_ptr + pid_e + 1)
    num_tokens = end_idx - start_idx

    row_start = pid_m * BLOCK_M
    if row_start >= num_tokens:
        return

    m_offs = row_start + tl.arange(0, BLOCK_M)
    m_mask = m_offs < num_tokens

    n_offs = pid_n * BLOCK_N + tl.arange(0, BLOCK_N)
    n_mask = n_offs < H
    n_block_idx = pid_n  # Since BLOCK_N=128, each pid_n maps to exactly one H-block

    acc = tl.zeros((BLOCK_M, BLOCK_N), dtype=tl.float32)

    k_range = tl.arange(0, BLOCK_K)
    row_indices = (start_idx + m_offs).to(tl.int64)

    # Precompute W2 base pointers for this (expert, n_tile)
    w2_base = (
        W2_ptr
        + pid_e.to(tl.int64) * stride_w2e
        + n_offs[None, :] * stride_w2n
    )
    w2scale_base = (
        W2_scale_ptr
        + pid_e.to(tl.int64) * stride_w2scale_e
        + n_block_idx * stride_w2scale_n
    )

    for k_start in range(0, I, BLOCK_K):
        k_block_idx = k_start // BLOCK_K
        k_offs = k_start + k_range

        # Load x1, x2 from G1 (float32)
        x1_ptrs = G1_ptr + row_indices[:, None] * stride_g1m + k_offs[None, :] * stride_g1k
        x2_ptrs = G1_ptr + row_indices[:, None] * stride_g1m + (I + k_offs)[None, :] * stride_g1k

        x1 = tl.load(x1_ptrs, mask=m_mask[:, None], other=0.0)
        x2 = tl.load(x2_ptrs, mask=m_mask[:, None], other=0.0)

        # SwiGLU: silu(x2) * x1 in float32
        c = x1 * x2 * tl.sigmoid(x2)  # shape [BLOCK_M, BLOCK_K], float32

        # ===== KEY CHANGE: Per-tile scalar quantization to FP8 =====
        # Use single scalar max for the entire tile (NOT per-row)
        c_abs_max = tl.max(tl.abs(c))  # scalar
        c_scale = tl.maximum(c_abs_max / 448.0, 1e-12)  # scalar, prevents div-by-zero
        c_fp8 = (c / c_scale).to(tl.float8_e4m3fn)  # [BLOCK_M, BLOCK_K] fp8

        # Load W2 tile (FP8)
        w2_ptrs = w2_base + k_offs[:, None] * stride_w2k
        w2_fp8 = tl.load(w2_ptrs, mask=n_mask[None, :], other=0.0)  # [BLOCK_K, BLOCK_N]

        # Load scalar W2 scale for this (n_block, k_block) tile
        w2_scale = tl.load(w2scale_base + k_block_idx * stride_w2scale_k)  # scalar

        # FP8 tensor core dot — both inputs are FP8 -> uses FP8 TCs on B200
        dot_result = tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32)  # [BLOCK_M, BLOCK_N]

        # Apply BOTH scales after the dot (mathematically exact)
        acc += dot_result * c_scale * w2_scale
        # ===== END KEY CHANGE =====

    # Apply routing weights
    routing_w = tl.load(
        Routing_weights_ptr + row_indices, mask=m_mask, other=0.0
    )
    acc = acc * routing_w[:, None]

    # Store output
    out_ptrs = (
        Out_ptr
        + row_indices[:, None] * stride_outm
        + n_offs[None, :] * stride_outn
    )
    tl.store(out_ptrs, acc, mask=m_mask[:, None] & n_mask[None, :])
```

**Step 3.2: Critical correctness of n_block_idx computation**

In the new kernel, we use `n_block_idx = pid_n` since `BLOCK_N = 128` (each program processes exactly one 128-block of the H dimension). This is cleaner and avoids the `n_offs // 128` computation inside the loop. The W2_scale lookup becomes:
```python
# w2scale_base already includes pid_n * stride_w2scale_n
w2_scale = tl.load(w2scale_base + k_block_idx * stride_w2scale_k)  # scalar float32
```

**Step 3.3: Verify the scale broadcast is correct**

For a single K-block iteration with FP8 TC:
- `c_fp8` has shape `[BLOCK_M, BLOCK_K]` — FP8
- `w2_fp8` has shape `[BLOCK_K, BLOCK_N]` — FP8 (W2 is [H, I], so [N, K] in the GEMM sense)
- `dot_result = tl.dot(c_fp8, w2_fp8)` → `[BLOCK_M, BLOCK_N]` float32
- `c_scale` is a scalar (single value)
- `w2_scale` is a scalar (single value for this tile)
- `acc += dot_result * c_scale * w2_scale` → broadcasts scalar multiplication over entire [BLOCK_M, BLOCK_N] tile

This is EXACTLY what the block-scale convention requires:
```
O_tile[m,n] = sum_k(c[m,k] * W2[k,n]) * W2_scale[n_block, k_block]
            = sum_k(c_fp8[m,k] * c_scale * W2[k,n]) * W2_scale
            = dot(c_fp8, W2)[m,n] * c_scale * W2_scale  ✓
```

The 4x speedup from FP8 TCs (vs FP32 TCs in the parent) is now unlocked for GEMM2.

### Phase 5: Complete `run()` Function

```python
@torch.no_grad()
def run(
    routing_logits, routing_bias,
    hidden_states, hidden_states_scale,
    gemm1_weights, gemm1_weights_scale,
    gemm2_weights, gemm2_weights_scale,
    local_expert_offset, routed_scaling_factor,
):
    H = 7168
    I = 2048
    device = hidden_states.device
    T = routing_logits.shape[0]
    E_local = gemm1_weights.shape[0]

    # Step 1: Route tokens (unchanged from parent)
    routed = _route_tokens(
        routing_logits, routing_bias,
        local_expert_offset, E_local, routed_scaling_factor,
    )
    if routed is None:
        return torch.zeros((T, H), dtype=torch.bfloat16, device=device)

    sorted_tokens, sorted_experts, sorted_weights, expert_counts, expert_offsets = routed
    Tk_total = sorted_tokens.numel()

    if Tk_total == 0:
        return torch.zeros((T, H), dtype=torch.bfloat16, device=device)

    max_count = int(expert_counts.max().item())
    if max_count == 0:
        return torch.zeros((T, H), dtype=torch.bfloat16, device=device)

    # Step 2: GEMM1 — unchanged, proven correct
    G1 = torch.zeros((Tk_total, 2 * I), dtype=torch.float32, device=device)
    grid_gemm1 = lambda META: (
        triton.cdiv(2 * I, META["BLOCK_N"]),
        triton.cdiv(max_count, META["BLOCK_M"]),
        E_local,
    )
    grouped_gemm1_kernel[grid_gemm1](
        A_ptr=hidden_states,
        A_scale_ptr=hidden_states_scale,
        W_ptr=gemm1_weights,
        W_scale_ptr=gemm1_weights_scale,
        Out_ptr=G1,
        sorted_tokens_ptr=sorted_tokens,
        expert_offsets_ptr=expert_offsets,
        H=H, N_dim=2 * I, MAX_TK=max_count,
        stride_am=hidden_states.stride(0),
        stride_ak=hidden_states.stride(1),
        stride_ascale_k=hidden_states_scale.stride(0),
        stride_ascale_m=hidden_states_scale.stride(1),
        stride_we=gemm1_weights.stride(0),
        stride_wn=gemm1_weights.stride(1),
        stride_wk=gemm1_weights.stride(2),
        stride_wscale_e=gemm1_weights_scale.stride(0),
        stride_wscale_n=gemm1_weights_scale.stride(1),
        stride_wscale_k=gemm1_weights_scale.stride(2),
        stride_outm=G1.stride(0), stride_outn=G1.stride(1),
        BLOCK_K=128,
    )

    # Step 3: GEMM2 with FP8 TCs — new kernel replaces parent's FP32 version
    O = torch.zeros((Tk_total, H), dtype=torch.float32, device=device)
    grid_gemm2 = lambda META: (
        triton.cdiv(H, META["BLOCK_N"]),  # H/128 = 56 tiles
        triton.cdiv(max_count, META["BLOCK_M"]),
        E_local,
    )
    grouped_gemm2_fp8_kernel[grid_gemm2](
        G1_ptr=G1,
        W2_ptr=gemm2_weights,
        W2_scale_ptr=gemm2_weights_scale,
        Routing_weights_ptr=sorted_weights,
        Out_ptr=O,
        expert_offsets_ptr=expert_offsets,
        I=I, H=H, MAX_TK=max_count,
        stride_g1m=G1.stride(0), stride_g1k=G1.stride(1),
        stride_w2e=gemm2_weights.stride(0),
        stride_w2n=gemm2_weights.stride(1),
        stride_w2k=gemm2_weights.stride(2),
        stride_w2scale_e=gemm2_weights_scale.stride(0),
        stride_w2scale_n=gemm2_weights_scale.stride(1),
        stride_w2scale_k=gemm2_weights_scale.stride(2),
        stride_outm=O.stride(0), stride_outn=O.stride(1),
        BLOCK_K=128,
    )

    # Step 4: Scatter-add (unchanged)
    output = torch.zeros((T, H), dtype=torch.float32, device=device)
    output.index_add_(0, sorted_tokens.long(), O)
    return output.to(torch.bfloat16)
```

### Phase 6: Safety Valve & Numerical Guarantees

**Step 6.1: FP8 clamp to prevent NaN/Inf**

The cast `.to(tl.float8_e4m3fn)` will clamp implicitly, but we can be explicit:
```python
# In the kernel, before casting:
c_clamped = tl.where(tl.abs(c / c_scale) > 448.0, tl.sign(c) * 448.0, c / c_scale)
c_fp8 = c_clamped.to(tl.float8_e4m3fn)
```
Actually, Triton's cast handles saturation automatically for fp8_e4m3fn, so the simpler form is:
```python
c_fp8 = (c / c_scale).to(tl.float8_e4m3fn)  # saturating cast, no NaN
```

**Step 6.2: Verify c_scale > 0**
```python
c_abs_max = tl.max(tl.abs(c))  # scalar
c_scale = tl.maximum(c_abs_max / 448.0, 1e-12)  # prevents division by zero
```

**Step 6.3: Integer type safety**
- All pointer arithmetic uses `.to(tl.int64)` for expert dimension offsets
- `pid_e.to(tl.int64) * stride_w2e` prevents int32 overflow for [32, 7168, 2048] tensors

**Step 6.4: Zeros initialization**
- `G1 = torch.zeros(...)` and `O = torch.zeros(...)` ensure masked rows contribute 0 to output
