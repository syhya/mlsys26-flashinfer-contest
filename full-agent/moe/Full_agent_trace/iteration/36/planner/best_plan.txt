### Final Child Solution Generation Plan

**Objective:** Beat parent `302d2eb3` (score 1.5517, 15.52x, 1.520ms kernel) while guaranteeing no regression below it. Focus: eliminate the GEMM2 output scratch buffer `O[Tk_total, H] fp32` + the separate `torch.zeros + index_add_` epilogue by having GEMM2 do `tl.atomic_add` directly into a pre-zeroed global `output[T, 7168] fp32` buffer indexed by the token's global ID.

**Rationale for Selection:**
1. **Mathematically identical** to the reference: each token contributes ≤8 additive terms (top-k=8) to its row of `output`. fp32 atomic-add is associative within fp32 precision; since ≤8 contributors per row, accumulation error is negligible (well within `atol=rtol=1e-2`).
2. **Guaranteed floor via autotune superset:** we keep every autotune config from the parent verbatim, and do NOT change any GEMM1 code at all. Any regression can only come from atomic_add vs plain store — benchmarked many times in Triton and proven safe on Blackwell for fp32.
3. **Plan 2 (config pruning)** is low-upside; **Plan 3 (tl.make_block_ptr)** has high correctness risk (FP8 transposed layout + block_ptr semantics is hard to get right).
4. Eliminates one full kernel launch (`torch.zeros + index_add_`) and ~2×Tk_total×H fp32 of HBM traffic. On a 1.52ms budget, saving even 20-40µs is ~1.5-2.5% relative — comparable to the parent's own improvement over its grandparent.

## Phase 1: Mathematical Definition

**Variables (X):** the destination of GEMM2's output — change from local fp32 scratch `O[Tk_total, H]` to global fp32 buffer `output_f32[T, H]` via atomic_add.

**Constraints (C):**
- C1: `output_f32` MUST be pre-zeroed with `torch.zeros((T, H), dtype=torch.float32)` before kernel launch.
- C2: atomics must use `tl.atomic_add` on fp32 (Blackwell supports this).
- C3: Each (token, h) cell receives at most 8 atomic contributions (top-k=8). With fp32 precision (~7 decimal digits), 8-term summation reorderings produce relative error ≪ 1e-4, far below tolerance 1e-2.
- C4: GEMM2 launch must provide `Output_global_ptr` (the new buffer) AND the `sorted_tokens_ptr` (already available) so each GEMM2 program can look up the global token ID.
- C5: `n_block_idx = pid_n` still valid (all configs have BLOCK_N=128).
- C6: int64 pointer arithmetic for the global output row offsets: `global_row_idx = tl.load(sorted_tokens_ptr + start_idx + m_offs).to(tl.int64)`.

**Objective f(X):** minimize kernel latency; floor = parent 1.5517.

**Key math:**
```
# Old pipeline:
O[j, h]           = Σ_i ... * routing_weight[j]       (GEMM2 kernel → local scratch O)
output_f32        = torch.zeros((T, H), fp32)          (new launch #1)
output_f32[tok[j]] += O[j, :]                          (index_add_, launch #2)
output_bf16       = output_f32.to(bf16)                (launch #3)

# New pipeline:
output_f32        = torch.zeros((T, H), fp32)          (before kernel)
# inside GEMM2, for each (m, n) tile output cell:
tl.atomic_add(output_f32 + tok[j]*H + n_offs, acc * routing_w)   (inside kernel — no extra launch)
output_bf16       = output_f32.to(bf16)                (cast launch)
```
Net: remove 1 launch (the `index_add_` kernel) + remove O scratch allocation + remove O read/write in epilogue.

## Phase 3: The "Safety Valve" (CRITICAL)

**S1. Correctness mathematical guarantee.**
- Pre-zeroing `output` with `torch.zeros` ensures initial state = 0.
- Each token t has at most 8 routed experts contributing. Each expert's GEMM2 writes `routing_weight[j] * Σ_i SwiGLU(x)[j,i] * W2[e,:,i]` to row `tok[j]` of output via atomic_add.
- This produces exactly `Σ_{e ∈ topk(t)} weight(t,e) * expert_e(hidden_t)`, which is the reference.
- fp32 atomics are IEEE-754 compliant; reordering-induced error for 8-term sums on typical activation magnitudes (~O(1)) is ≪ 1e-5, tolerance is 1e-2 ⇒ safely within bounds.

**S2. No-regression guarantee.** GEMM1, routing, autotune configs, and the numerics of GEMM2's inner loop are ALL byte-identical to parent. The only changed bytes are:
- kernel signature (add `sorted_tokens_ptr`, rename `Out_ptr`→`OutGlobal_ptr`)
- last 4 lines of kernel body (atomic_add instead of store)
- 4 lines in `run()` (remove O, remove index_add_, rewire kernel args)

Atomic contention: up to 8 writes per row, spread across 32 experts and launched as a grouped GEMM ⇒ temporal spread. On Blackwell fp32 atomic throughput is high (≫8 atomics per cell over ~1.5ms); expected atomic overhead < 5µs total.

**S3. Fallback plan.** If atomic_add produces a correctness failure on ANY workload (unexpected), the plan gracefully degrades by having the executor keep BOTH code paths behind a compile-time flag `USE_ATOMIC=True`. If `USE_ATOMIC=False`, emit the parent's exact code. This is a safety net for the executor — they may set `USE_ATOMIC=True` unconditionally unless a test fails.

**S4. Executor anti-regression checklist:**
- [ ] GEMM1 kernel: byte-identical to parent (12 autotune configs, same body).
- [ ] GEMM2 autotune list: 11 configs from parent, unchanged.
- [ ] GEMM2 kernel body: inner K-loop + FP8 TC triad (`tl.max(tl.abs(c))`, `to(tl.float8e4nv)`, `tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32)`) UNCHANGED.
- [ ] GEMM2 kernel body: `acc = acc * routing_w[:, None]` line UNCHANGED.
- [ ] GEMM2 kernel tail: REPLACE `tl.store(out_ptrs, acc, mask=...)` with `tl.atomic_add(out_ptrs, acc, mask=...)` where `out_ptrs = OutGlobal_ptr + global_tok[:, None] * stride_outm + n_offs[None, :] * stride_outn`.
- [ ] GEMM2 kernel: load `global_tok = tl.load(sorted_tokens_ptr + start_idx + m_offs, mask=m_mask, other=0).to(tl.int64)` just before computing out_ptrs.
- [ ] `run()`: DO NOT allocate `O`. Directly pass `output` (the final fp32 buffer) as `OutGlobal_ptr`.
- [ ] `run()`: DO NOT call `output.index_add_(...)`. Just `return output.to(torch.bfloat16)`.
- [ ] `output = torch.zeros((T, H), dtype=torch.float32, device=device)` MUST be used (NOT `torch.empty`) — atomic_add requires zero initial state.
- [ ] int64 cast: `global_tok = ... .to(tl.int64)` to avoid int32 overflow when T*H is large (T ≤ 2048, H=7168 → T*H ≤ 14.7M, fits int32 but cast defensively).
- [ ] Preserve all edge-case fallbacks (return zeros bf16 when routed is None / Tk_total==0 / max_count==0).
