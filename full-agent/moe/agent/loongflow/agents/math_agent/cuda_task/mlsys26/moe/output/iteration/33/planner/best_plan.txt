## Phase 1: Mathematical Definition

**Variables:**
- `T`: token count; `E_local=32`; `H=7168`; `I=2048`
- Tile dims: `BLOCK_M ∈ {16,32,64,128,256}`, `BLOCK_N=128`, `BLOCK_K=128`
- Per-tile scalar quantization: `c_scale = max(|c|)/448.0` (NO axis arg)
- `c_fp8 = (c/c_scale).to(tl.float8e4nv)`

**Complete MoE math (preserved from 8e3c776e):**
```
G1[j,n] = Σ_k( hidden[tok[j],k] * W1[e,n,k] ) * h_scale[k//128,tok[j]] * w1_scale[e,n//128,k//128]
C[j,i]  = G1[j,i] * G1[j,I+i] * sigmoid(G1[j,I+i])
O[j,h]  = Σ_i( C_fp8[j,i]*c_scale * W2[e,h,i]) * w2_scale[e,h//128,i//128]
       = dot(C_fp8, W2_fp8)[j,h] * c_scale * w2_scale   (both scalars → exact)
Y[tok[j],h] += routing_weight[j] * O[j,h]
```

**Constraints (hard, must hold):**
- C1: `BLOCK_N=128` for ALL GEMM2 configs (enables `n_block_idx=pid_n` shortcut)
- C2: `BLOCK_K=128` strictly (one K-tile = one 128-block scale)
- C3: `pid_e.to(tl.int64)` before every stride multiply (int32 overflow prevention)
- C4: `tl.max(tl.abs(c))` NO axis argument (scalar, not vector)
- C5: `torch.zeros` (not `torch.empty`) for G1, O, output
- C6: `(start_idx + m_offs).to(tl.int64)` for output row offsets

## Phase 3: The "Safety Valve" (CRITICAL)

**S1. Floor guarantee via config preservation.** All 13 GEMM1 + 12 GEMM2 configs from `8e3c776e` are preserved verbatim. The Triton autotuner always benchmarks all configs for the given `MAX_TK` key and selects the minimum-latency one. Therefore kernel_latency ≤ latency_of(best 8e3c776e config) for every workload. Floor: score ≥ 1.5338 (minus ~0.01 benchmark noise).

**S2. Correctness-preserving kernel body.** We do NOT modify any kernel body — only the `configs=[...]` list inside `@triton.autotune`. Triton autotune configs are semantically transparent: they cannot cause correctness failures, only performance differences or compile-time errors (which cause the autotuner to silently drop that config from the pool).

**S3. Compile-failure safety.** If either of the 2 new configs fails to compile (e.g., register spill at BLOCK_M=256,num_stages=4 due to insufficient SMEM on some architectures), Triton's autotune mechanism silently excludes them and selects from the remaining configs. There is NO crash path.

- [ ] GEMM2 configs list: 12 original + 2 new = 14 configs
- [ ] GEMM2 kernel body: uses `tl.float8e4nv` (NOT `tl.float8_e4m3fn`)
- [ ] `c_abs_max = tl.max(tl.abs(c))` — NO axis argument
- [ ] `tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32)` — both operands FP8
- [ ] K-loop uses `tl.range(0, I, BLOCK_K)` and `tl.range(0, H, BLOCK_K)` (NOT plain range)
