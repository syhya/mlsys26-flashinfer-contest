*   **Routing Logic Correction (The 8192.0 Error Fix)**: Previous iterations incorrectly normalized routing weights using `weights / weights.sum(dim=1)`. DeepSeek-V3 explicitly uses the unnormalized sigmoid values for the selected Top-K experts. The solver must calculate: `weights = s * M * routed_scaling_factor` (where `M` is the Top-K mask) and completely delete the division by `weights_sum`.
*   **Precision Safety Valve (BF16 Upcasting)**: To ensure numerical stability in the SwiGLU activation and GEMM2, upcast the SwiGLU output and the dequantized weights to `bfloat16`. 
    *   *Formula*: `w_bf16 = (w.to(tl.float32) * w_scale).to(tl.bfloat16)` and `c_bf16 = c.to(tl.bfloat16)`.
*   **Pointer Overflow Safety**: Enforce `.to(tl.int64)` on all expert indices (`e_idx`) and coordinate offsets *before* multiplying them by tensor strides to avoid $2^{31}-1$ int32 overflow.
