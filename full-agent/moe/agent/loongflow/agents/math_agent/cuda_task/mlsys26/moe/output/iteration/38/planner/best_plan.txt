## Phase 1: Mathematical Definition

**Pipeline math (unchanged from `302d2eb3`):**
```
G1[j,n] = Σ_k hidden[tok[j],k]·W1[e,n,k] · h_scale[k//128,tok[j]] · w1_scale[e,n//128,k//128]
C[j,i]  = G1[j,i] · G1[j,I+i] · sigmoid(G1[j,I+i])         # SwiGLU
O[j,h]  = Σ_i C[j,i] · W2[e,h,i] · w2_scale[e,h//128,i//128]
output[tok[j],h] += routing_w[j] · O[j,h]                  # ≤8 contributions per row
```

**Variables (X):**
- Final accumulator dtype (CHANGED: bfloat16 instead of float32)
- All other components: UNCHANGED

**Constraints (C, hard):**
- C1: `output = torch.zeros((T,H), dtype=torch.bfloat16, device=device)` — must be zeros (not empty), must be bf16.
- C2: `O.to(torch.bfloat16)` cast must precede `index_add_` (PyTorch requires matching dtypes).
- C3: All GEMM kernels and their autotune lists byte-identical to `302d2eb3`.
- C4: `_route_tokens` byte-identical to `302d2eb3`'s sentinel-sort version.
- C5: All BLOCK_N=128 in GEMM2 configs (preserved from `302d2eb3`).
- C6: int64 pointer arithmetic preserved everywhere strides are multiplied by `pid_e`.

**Bf16 accumulation error analysis:** Each output row receives ≤ TOP_K=8 contributions of magnitude ~O(routed_scaling_factor). bf16 has ~3 decimal digits; 8-term accumulation error is bounded by ~8 × 2⁻⁷ ≈ 0.06 in worst case but typically ~1e-3. Tolerance is rtol=atol=1e-2. The lineage has already accepted this in `ba626bb8` (passed correctness with max_abs_err=34,816 on outputs of magnitude ~32k).

**Objective:** minimize end-to-end latency.

in_range = (shifted >= 0) & (shifted < e_local)
sentinel = torch.full_like(shifted, e_local)
sort_key = torch.where(in_range, shifted, sentinel)
order = torch.argsort(sort_key, stable=True)
sort_key_sorted = sort_key[order]
num_valid_t = (sort_key_sorted < e_local).sum()
num_valid = int(num_valid_t.item())
if num_valid == 0:

valid_order = order[:num_valid]
sorted_tokens  = flat_token[valid_order].to(torch.int32)
sorted_experts = shifted[valid_order].to(torch.int32)
sorted_weights = flat_weight[valid_order].to(torch.float32)
expert_counts = torch.bincount(sorted_experts, minlength=e_local)
expert_offsets = torch.zeros(e_local + 1, dtype=torch.int32, device=device)
expert_offsets[1:] = torch.cumsum(expert_counts, dim=0)
