## Best Plan:

### Phase 1: Mathematical Definition

**Step 1.1: Complete MoE pipeline math (unchanged):**
```
G1[j,n] = Σ_k hidden[token[j],k]·W1[e,n,k] · h_scale[k//128,token[j]] · w1_scale[e,n//128,k//128]
C[j,i]  = G1[j,i] · G1[j,I+i] · sigmoid(G1[j,I+i])    # SwiGLU
O[j,h]  = Σ_i C[j,i] · W2[e,h,i] · w2_scale[e,h//128,i//128]
output[token[j],h] += routing_weight[j] · O[j,h]
```

**Step 1.2: Per-tile SCALAR FP8 quantization (the GEMM2 key innovation, proven by a96d6608):**
```
c_abs_max = tl.max(tl.abs(c))                  # SCALAR — NO axis arg!
c_scale   = tl.maximum(c_abs_max / 448.0, 1e-12)
c_fp8     = (c / c_scale).to(tl.float8e4nv)    # use float8e4nv alias (a96d6608 proved it works)
acc      += tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32) * c_scale * w2_scale
```
Both c_scale and w2_scale are scalars → exact post-dot dequant.

CRITICAL: `tl.max(tl.abs(c))` MUST have NO axis argument (scalar). Using `axis=1` produces a vector and SILENTLY breaks correctness.

### Phase 3: GEMM2 Kernel — Copy a96d6608's FP8 TC kernel verbatim, expand autotune

EXECUTOR DIRECTIVE: Copy the EXACT GEMM2 kernel body from solution `a96d6608`. Do NOT substitute BF16 or FP32 paths. The cast `(c / c_scale).to(tl.float8e4nv)` is PROVEN to work in this eval environment (a96d6608 passed correctness with score=1.5338).

```python
@triton.autotune(
    configs=[
        triton.Config({"BLOCK_M": 16,  "BLOCK_N": 128}, num_warps=4,  num_stages=4),
        triton.Config({"BLOCK_M": 32,  "BLOCK_N": 128}, num_warps=4,  num_stages=4),
        triton.Config({"BLOCK_M": 32,  "BLOCK_N": 128}, num_warps=4,  num_stages=5),
        triton.Config({"BLOCK_M": 64,  "BLOCK_N": 128}, num_warps=4,  num_stages=3),
        triton.Config({"BLOCK_M": 64,  "BLOCK_N": 128}, num_warps=4,  num_stages=4),
        triton.Config({"BLOCK_M": 64,  "BLOCK_N": 128}, num_warps=4,  num_stages=5),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=8,  num_stages=3),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=8,  num_stages=4),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=8,  num_stages=5),
        triton.Config({"BLOCK_M": 128, "BLOCK_N": 128}, num_warps=16, num_stages=4),
        # NEW: BLOCK_M=256 for large-prefill workloads
        triton.Config({"BLOCK_M": 256, "BLOCK_N": 128}, num_warps=8,  num_stages=3),
        triton.Config({"BLOCK_M": 256, "BLOCK_N": 128}, num_warps=16, num_stages=3),
    ],
    key=["MAX_TK"],
)
@triton.jit
def grouped_gemm2_kernel(
    G1_ptr, W2_ptr, W2_scale_ptr, Routing_weights_ptr, Out_ptr,
    expert_offsets_ptr,
    I: tl.constexpr, H: tl.constexpr, MAX_TK,
    stride_g1m, stride_g1k,
    stride_w2e, stride_w2n, stride_w2k,
    stride_w2scale_e, stride_w2scale_n, stride_w2scale_k,
    stride_outm, stride_outn,
    BLOCK_K: tl.constexpr, BLOCK_M: tl.constexpr, BLOCK_N: tl.constexpr,
):
    pid_n = tl.program_id(0); pid_m = tl.program_id(1); pid_e = tl.program_id(2)
    start_idx = tl.load(expert_offsets_ptr + pid_e)
    end_idx   = tl.load(expert_offsets_ptr + pid_e + 1)
    num_tokens = end_idx - start_idx
    if pid_m * BLOCK_M >= num_tokens:
        return

    m_offs = pid_m * BLOCK_M + tl.arange(0, BLOCK_M)
    m_mask = m_offs < num_tokens
    n_offs = pid_n * BLOCK_N + tl.arange(0, BLOCK_N)
    n_mask = n_offs < H
    n_block_idx = pid_n  # valid because all configs have BLOCK_N=128

    acc = tl.zeros((BLOCK_M, BLOCK_N), dtype=tl.float32)
    k_range = tl.arange(0, BLOCK_K)
    row_indices = (start_idx + m_offs).to(tl.int64)

    # Hoist base pointers OUTSIDE the K-loop
    w2_base = W2_ptr + pid_e.to(tl.int64) * stride_w2e + n_offs[None, :] * stride_w2n
    w2scale_base = W2_scale_ptr + pid_e.to(tl.int64) * stride_w2scale_e + n_block_idx * stride_w2scale_n

    # tl.range() enables software pipelining (proven +12% by lineage)
    for k_start in tl.range(0, I, BLOCK_K):
        k_block_idx = k_start // BLOCK_K
        k_offs = k_start + k_range

        x1_ptrs = G1_ptr + row_indices[:, None] * stride_g1m + k_offs[None, :] * stride_g1k
        x2_ptrs = G1_ptr + row_indices[:, None] * stride_g1m + (I + k_offs)[None, :] * stride_g1k
        x1 = tl.load(x1_ptrs, mask=m_mask[:, None], other=0.0)
        x2 = tl.load(x2_ptrs, mask=m_mask[:, None], other=0.0)

        # Fused SwiGLU: silu(x2) * x1
        c = x1 * x2 * tl.sigmoid(x2)

        # === KEY: per-tile SCALAR FP8 quantization (NO axis arg!) ===
        c_abs_max = tl.max(tl.abs(c))                 # SCALAR
        c_scale   = tl.maximum(c_abs_max / 448.0, 1e-12)
        c_fp8     = (c / c_scale).to(tl.float8e4nv)   # alias for fp8_e4m3fn

        # Load W2 FP8 tile
        w2_ptrs = w2_base + k_offs[:, None] * stride_w2k
        w2_fp8  = tl.load(w2_ptrs, mask=n_mask[None, :], other=0.0)

        # SCALAR W2 scale
        w2_scale = tl.load(w2scale_base + k_block_idx * stride_w2scale_k)

        # FP8 Tensor Core dot — both operands FP8
        dot_result = tl.dot(c_fp8, w2_fp8, out_dtype=tl.float32)

        # Exact post-dot dequant (both scalars)
        acc += dot_result * c_scale * w2_scale

    routing_w = tl.load(Routing_weights_ptr + row_indices, mask=m_mask, other=0.0)
    acc = acc * routing_w[:, None]

    out_ptrs = Out_ptr + row_indices[:, None] * stride_outm + n_offs[None, :] * stride_outn
    tl.store(out_ptrs, acc, mask=m_mask[:, None] & n_mask[None, :])
```

### Phase 5: Safety Valve — Mandatory Correctness Checks

1. **NO axis on tl.max**: `tl.max(tl.abs(c))` — scalar reduction. Per-row would silently break math.
2. **All BLOCK_N=128**: Verified for all 12 GEMM2 configs → `n_block_idx = pid_n` valid.
3. **int64 pointer arithmetic**: `pid_e.to(tl.int64) * stride_w2e` (gemm2_weights[32,7168,2048] stride[0]=14,680,064 exceeds int32 for pid_e>=146). All pointer math involving pid_e MUST cast to int64.
4. **torch.zeros (not torch.empty)** for G1, O, output buffers — masked threads/early-exits must contribute 0.
5. **Edge-case fallbacks**: Return zeros((T,H), bf16) if routed is None / Tk_total==0 / max_count==0.
6. **BLOCK_M=256 register pressure mitigation**: If autotune compilation fails for BLOCK_M=256 configs (excessive register spill), the autotuner will simply skip them — other 10 configs remain functional. NO crash risk.
7. **Use `tl.float8e4nv` alias**: a96d6608 proved this alias works; `tl.float8_e4m3fn` may fail in some Triton builds. Use the alias.
