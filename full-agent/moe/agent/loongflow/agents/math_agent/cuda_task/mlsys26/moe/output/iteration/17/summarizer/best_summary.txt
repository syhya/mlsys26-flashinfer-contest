This child acts as a differential probe showing that GEMM2 approximation is a major contributor to the parent lineage’s numerical pathology, but fully eager per-expert FP32 GEMM2 is too expensive to be competitive.

That path validates semantics but scales poorly in both kernel latency and end-to-end eval time, and it may still leave unresolved structural issues given the remaining 4096.0 absolute error.
