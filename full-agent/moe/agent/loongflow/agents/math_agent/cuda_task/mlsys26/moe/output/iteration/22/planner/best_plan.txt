## Phase 1: Mathematical Definition

### 1.1 Problem Variables

**Input Tensors:**
- `routing_logits ∈ R^{T×256}` (float32)
- `routing_bias ∈ R^{256}` (bfloat16)
- `hidden_states ∈ FP8^{T×H}` where H=7168
- `hidden_states_scale ∈ R^{(H/128)×T}` (transposed layout: feature-block × token)
- `gemm1_weights ∈ FP8^{E×(2I)×H}` where E=32, I=2048
- `gemm1_weights_scale ∈ R^{E×(2I/128)×(H/128)}`
- `gemm2_weights ∈ FP8^{E×H×I}`
- `gemm2_weights_scale ∈ R^{E×(H/128)×(I/128)}`

**Routing Decision Variables:**
- `s[t, e] = sigmoid(routing_logits[t, e])` (base routing scores)
- `sb[t, e] = s[t, e] + routing_bias[e]` (biased scores for selection)
- `group_scores[t, g] = sum of top-2 sb values in group g`
- `selected_groups[t] ⊆ {0..7}` (top-4 groups, |selected_groups[t]| = 4)
- `topk_idx[t, j] ∈ {0..255}` (top-8 expert indices, j ∈ {0..7})
- `topk_w[t, j] ∈ R` (normalized routing weights from s, not sb)

**Dispatch Variables:**
- `sorted_tokens[k] ∈ {0..T-1}` (global token index for dispatch row k)
- `sorted_experts[k] ∈ {0..31}` (local expert index for dispatch row k)
- `sorted_weights[k] ∈ R` (routing weight for dispatch row k)
- `expert_offsets[e] ∈ Z` (CSR offset array, length E+1)

**Computation Variables:**
- `A_e ∈ FP8^{T_e×H}` (gathered activation rows for expert e)
- `A_scale_e ∈ R^{T_e×(H/128)}` (gathered activation scales)
- `G1_e ∈ R^{T_e×2I}` (GEMM1 output, FP32 accumulation)
- `x1_e = G1_e[:, :I]`, `x2_e = G1_e[:, I:]`
- `S_e = silu(x2_e) * x1_e ∈ R^{T_e×I}` (SwiGLU output)
- `Y_e ∈ R^{T_e×H}` (GEMM2 output)
- `Out ∈ R^{T×H}` (final accumulated output, FP32)

### 1.2 Hard Constraints

**C1. Exact Routing Topology:**
1. Compute `s = sigmoid(routing_logits.float())`
2. Compute `sb = s + routing_bias.float()`
3. Reshape `sb` into `[T, 8, 32]` (8 groups, 32 experts per group)
4. For each group: `top2_vals = topk(sb[:, g, :], k=2, dim=1).values`
5. Compute `group_scores[:, g] = sum(top2_vals, dim=1)`
6. Select `top_groups = topk(group_scores, k=4, dim=1).indices`
7. Mask experts outside selected groups: `expert_mask[t, e] = (e//32 in top_groups[t])`
8. Select `topk_idx = topk(sb.masked_fill(~expert_mask, -inf), k=8, dim=1).indices`
9. Compute final weights from **original s** (not sb): `topk_w = s.gather(1, topk_idx)`
10. Normalize: `topk_w = topk_w / (topk_w.sum(dim=1, keepdim=True) + 1e-20)`
11. Scale: `topk_w *= routed_scaling_factor`

**C2. Exact Block-Scale Dequantization:**
- Activation: `A_deq[t, k] = hidden_states[t, k] * hidden_states_scale[k//128, t]`
- GEMM1 weight: `W1_deq[e, n, k] = gemm1_weights[e, n, k] * gemm1_weights_scale[e, n//128, k//128]`
- GEMM2 weight: `W2_deq[e, h, i] = gemm2_weights[e, h, i] * gemm2_weights_scale[e, h//128, i//128]`

**C3. FP32 Accumulation:**
- GEMM1 and GEMM2 must accumulate in FP32
- Only final output converts to BF16

**C4. Exact Dispatch:**
- Every (token, expert) pair from top-8 routing must be processed exactly once
- Scatter-add must sum all expert contributions for each token

**C5. Output Specification:**
- Shape: `[T, 7168]`
- Dtype: `torch.bfloat16`
- Numerical tolerance: `rtol=1e-2, atol=1e-2`

### 1.3 Objective Function

Lexicographic optimization:
1. **Primary:** Satisfy correctness constraints C1-C5 (mandatory)
2. **Secondary:** Minimize execution time

Formally: `Score = Correctness_Pass × SpeedFactor`

## Phase 2: The Optimization Loop

This is a deterministic execution strategy, not a stochastic search.

### 2.1 Routing (Exact, Vectorized PyTorch)

```python
def route_tokens(routing_logits, routing_bias, routed_scaling_factor):
    T, E_global = routing_logits.shape  # E_global = 256
    
    # Step 1: Base sigmoid scores
    s = torch.sigmoid(routing_logits.float())
    
    # Step 2: Biased scores for selection
    sb = s + routing_bias.float().view(1, E_global)
    
    # Step 3: Reshape into groups [T, 8, 32]
    sbg = sb.view(T, 8, 32)
    
    # Step 4: Group top-2 sum
    top2_vals = torch.topk(sbg, k=2, dim=2, largest=True, sorted=False).values
    group_scores = top2_vals.sum(dim=2)  # [T, 8]
    
    # Step 5: Top-4 groups
    top_groups = torch.topk(group_scores, k=4, dim=1, largest=True, sorted=False).indices
    
    # Step 6: Build expert mask
    group_mask = torch.zeros((T, 8), device=routing_logits.device, dtype=torch.bool)
    group_mask.scatter_(1, top_groups, True)
    expert_mask = group_mask.unsqueeze(-1).expand(T, 8, 32).reshape(T, E_global)
    
    # Step 7: Mask and select top-8 experts
    pruned = sb.masked_fill(~expert_mask, float('-inf'))
    topk_idx = torch.topk(pruned, k=8, dim=1, largest=True, sorted=False).indices
    
    # Step 8: Final weights from original sigmoid (not biased)
    topk_w = s.gather(1, topk_idx)
    topk_w = topk_w / (topk_w.sum(dim=1, keepdim=True) + 1e-20)
    topk_w = topk_w * routed_scaling_factor
    
    return topk_idx, topk_w
```

**Correctness guarantee:** This exactly implements the DeepSeek routing specification using only deterministic PyTorch operations.

### 2.2 Dispatch Building (Exact, GPU-side)

```python
def build_dispatch(topk_idx, topk_w, local_expert_offset, E_local):
    device = topk_idx.device
    
    # Step 1: Filter local experts
    local_mask = (topk_idx >= local_expert_offset) & (topk_idx < local_expert_offset + E_local)
    tok, slot = torch.nonzero(local_mask, as_tuple=True)
    
    if tok.numel() == 0:
        return (
            torch.empty(0, device=device, dtype=torch.long),
            torch.empty(0, device=device, dtype=torch.long),
            torch.empty(0, device=device, dtype=torch.float32),
            torch.zeros(E_local + 1, device=device, dtype=torch.long)
        )
    
    # Step 2: Extract local expert indices and weights
    local_exp = (topk_idx[tok, slot] - local_expert_offset).long()
    weights = topk_w[tok, slot].float()
    
    # Step 3: Sort by expert for contiguous access
    order = torch.argsort(local_exp, stable=True)
    sorted_tokens = tok[order]
    sorted_experts = local_exp[order]
    sorted_weights = weights[order]
    
    # Step 4: Build CSR offsets
    counts = torch.bincount(sorted_experts, minlength=E_local)
    offsets = torch.zeros(E_local + 1, device=device, dtype=torch.long)
    offsets[1:] = torch.cumsum(counts, dim=0)
    
    return sorted_tokens, sorted_experts, sorted_weights, offsets
```

**Correctness guarantee:** Uses only exact PyTorch operations; no approximations or heuristics.

### 2.3 Per-Expert Execution with Native FP8 Backend

```python
def execute_expert_native_fp8(
    expert_idx, start, end,
    sorted_tokens, sorted_weights,
    hidden_states, hidden_states_scale,
    gemm1_weights, gemm1_weights_scale,
    gemm2_weights, gemm2_weights_scale
):
    """
    Attempt to use torch._scaled_mm for FP8 tensor core acceleration.
    Falls back to exact dequant if native API doesn't support layout.
    """
    
    T_e = end - start
    tok_e = sorted_tokens[start:end]
    w_e = sorted_weights[start:end]
    
    # Gather activations
    A_fp8 = hidden_states[tok_e]  # [T_e, 7168] fp8
    
    # Gather activation scales [56, T_e] -> [T_e, 56]
    A_scale = hidden_states_scale[:, tok_e].t().contiguous()
    
    # Try native FP8 GEMM1
    try:
        # torch._scaled_mm expects specific scale layouts
        # Attempt to reshape scales to match API requirements
        W1_fp8 = gemm1_weights[expert_idx]  # [4096, 7168] fp8
        W1_scale = gemm1_weights_scale[expert_idx]  # [32, 56]
        
        # Expand scales to per-element or per-block format as needed
        # This is the critical adaptation step
        A_scale_expanded = A_scale.repeat_interleave(128, dim=1)[:, :7168]
        W1_scale_expanded = W1_scale.repeat_interleave(128, dim=0).repeat_interleave(128, dim=1)[:4096, :7168]
        
        # Attempt native scaled matmul
        G1 = torch._scaled_mm(
            A_fp8, W1_fp8.t(),
            scale_a=A_scale_expanded,
            scale_b=W1_scale_expanded,
            out_dtype=torch.float32
        )
        
    except (RuntimeError, TypeError, AttributeError):
        # Fallback: exact dequant
        A_fp32 = A_fp8.float() * A_scale_expanded
        W1_fp32 = W1_fp8.float() * W1_scale_expanded
        G1 = torch.mm(A_fp32, W1_fp32.t())
    
    # SwiGLU
    x1 = G1[:, :2048]
    x2 = G1[:, 2048:]
    S = torch.nn.functional.silu(x2) * x1
    
    # Try native FP8 GEMM2
    try:
        W2_fp8 = gemm2_weights[expert_idx]  # [7168, 2048] fp8
        W2_scale = gemm2_weights_scale[expert_idx]  # [56, 16]
        
        W2_scale_expanded = W2_scale.repeat_interleave(128, dim=0).repeat_interleave(128, dim=1)[:7168, :2048]
        
        # S is fp32, need to handle mixed precision
        # Option 1: Keep S in fp32 and use standard matmul
        Y = torch.mm(S, W2_fp8.float() * W2_scale_expanded.t())
        
    except (RuntimeError, TypeError, AttributeError):
        # Fallback: exact dequant
        W2_fp32 = W2_fp8.float() * W2_scale_expanded
        Y = torch.mm(S, W2_fp32.t())
    
    # Apply routing weights
    Y = Y * w_e.unsqueeze(1)
    
    return tok_e, Y
```

**Correctness guarantee:** Fallback path uses exact blockwise dequantization, which is mathematically equivalent to the reference implementation.

### 2.4 Main Execution Loop

```python
def run(...):
    # Phase 1: Routing
    topk_idx, topk_w = route_tokens(routing_logits, routing_bias, routed_scaling_factor)
    
    # Phase 2: Dispatch
    sorted_tokens, sorted_experts, sorted_weights, expert_offsets = build_dispatch(
        topk_idx, topk_w, local_expert_offset, 32
    )
    
    # Phase 3: Initialize output
    output = torch.zeros((T, 7168), device=hidden_states.device, dtype=torch.float32)
    
    if sorted_tokens.numel() == 0:
        return output.to(torch.bfloat16)
    
    # Phase 4: Per-expert execution
    for e in range(32):
        start = expert_offsets[e].item()
        end = expert_offsets[e + 1].item()
        
        if start == end:
            continue
        
        tok_e, Y_e = execute_expert_native_fp8(
            e, start, end,
            sorted_tokens, sorted_weights,
            hidden_states, hidden_states_scale,
            gemm1_weights, gemm1_weights_scale,
            gemm2_weights, gemm2_weights_scale
        )
        
        # Phase 5: Accumulate
        output.index_add_(0, tok_e, Y_e)
    
    # Phase 6: Final conversion
    return output.to(torch.bfloat16)
```

### 3.2 Exact Dequantization Fallback

The fallback path MUST:
1. Convert FP8 to FP32: `fp8_tensor.float()`
2. Expand block scales using `repeat_interleave(128, dim=...)`
3. Element-wise multiply: `fp32_tensor * expanded_scale`

### 3.3 Numerical Stability Guards

3. **Empty batch handling:** Check `tok.numel() == 0` before processing
4. **Output range clamping:** Optional `torch.clamp` before BF16 conversion if needed

### 4.3 Scale Expansion Helper

```python
def expand_block_scales(scale_2d, block_size, target_shape):
    """
    Expand [n_blocks_dim0, n_blocks_dim1] to [dim0, dim1]
    where each block is block_size × block_size
    """
    expanded = scale_2d.repeat_interleave(block_size, dim=0).repeat_interleave(block_size, dim=1)
    return expanded[:target_shape[0], :target_shape[1]]
```
