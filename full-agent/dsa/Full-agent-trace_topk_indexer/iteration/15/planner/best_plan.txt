### Phase 1: Mathematical Definition

For batch `b`, `L = seq_lens[b]`, `K = 2048`, `P = 64`, `N = max_num_pages * 64`.

`score[b, p¬∑64 + t] = Œ£_h ReLU( scale[t] ¬∑ Œ£_d Q[b,h,d] ¬∑ fp8(K[t,d]) ) ¬∑ weights[b,h]`, `t < L`.

Output: any permutation of top-`min(K, L)` indices (global `block_table[b,p]¬∑64 + t`); rest = -1.

**Four-tier partition (mutually exclusive, exhaustive):**
| Tier | Condition | Mechanism |
|------|-----------|-----------|
| S    | `L ‚â§ 2048`                   | Direct enumeration (parent's `emit_short_topk_kernel`) |
| M    | `2048 < L ‚â§ 6144`            | `block_topk_kernel<256, 24>` (parent's) |
| **L2 (NEW)** | **`6144 < L ‚â§ 12288`** | **`block_topk_kernel_large<512, 24>`** |
| L    | `L > 12288`                  | Device-wide `cub::DeviceSegmentedRadixSort` (parent's path) |

### Phase 2: The Optimization Loop (Kernel Architecture)

**A. `emit_short_topk_kernel`** ‚Äî verbatim from parent. Handles TIER-S.

**B. `compute_scores_kernel`** ‚Äî verbatim from parent. Guard `if (seq_len <= 2048) return;` already present. Writes scores/indices into `scores_buf`/`indices_buf` at `[b*N + p*64 + t]`. Used by all of TIER-M, L2, L.

**C. `block_topk_kernel<256, 24>`** ‚Äî verbatim from parent. Guard `if (L <= 2048 || L > L_MED_MAX) return;`. Handles TIER-M.

**D. NEW `block_topk_kernel_large<512, 24>`** ‚Äî handles TIER-L2. Structure copied from `block_topk_kernel`, only parameters change:
```cpp
template <int BLOCK_THREADS, int ITEMS_PER_THREAD>
__global__ void block_topk_kernel_large(
    const float* __restrict__ scores_buf,
    const int32_t* __restrict__ indices_buf,
    const int32_t* __restrict__ seq_lens,
    int32_t* __restrict__ topk_indices,
    int N, int topk, int l_med_max, int l_l2_max)
{
    int b = blockIdx.x;
    int L = seq_lens[b];
    if (L <= l_med_max || L > l_l2_max) return;     // only TIER-L2

    using BlockRadixSort = cub::BlockRadixSort<float, BLOCK_THREADS, ITEMS_PER_THREAD, int32_t>;
    __shared__ typename BlockRadixSort::TempStorage temp_storage;

    float   keys[ITEMS_PER_THREAD];
    int32_t vals[ITEMS_PER_THREAD];

    int base = b * N;
    #pragma unroll
    for (int i = 0; i < ITEMS_PER_THREAD; i++) {
        int idx = i * BLOCK_THREADS + threadIdx.x;   // strided load
        if (idx < L) {
            keys[i] = scores_buf[base + idx];
            vals[i] = indices_buf[base + idx];
        } else {
            keys[i] = -1e30f;
            vals[i] = -1;
        }
    }

    BlockRadixSort(temp_storage).SortDescendingBlockedToStriped(keys, vals);

    // Strided write (matches BlockedToStriped output layout)
    #pragma unroll
    for (int i = 0; i < ITEMS_PER_THREAD; i++) {
        int out_pos = i * BLOCK_THREADS + threadIdx.x;
        if (out_pos < topk) {
            topk_indices[b * topk + out_pos] = vals[i];
        }
    }
}
```
Launch: `<<<batch_size, 512>>>`. **CRITICAL:** load pattern is strided (`i*BLOCK_THREADS + threadIdx.x`) and `SortDescendingBlockedToStriped` with strided output write ‚Äî identical pairing to parent's TIER-M kernel (this is a correctness-sensitive detail flagged by the parent summary).

**E. `generate_offsets_begin_end`** ‚Äî MODIFY: empty segment for all batches with `L ‚â§ L_L2_MAX`. CUB only processes TIER-L:
```cpp
int L = seq_lens[b];
begin_offsets[b] = b * segment_size;
end_offsets[b]   = (L > L_L2_MAX) ? (b * segment_size + L) : (b * segment_size);
```

**F. `extract_topk_kernel`** ‚Äî MODIFY: early-return if `seq_lens[b] <= L_L2_MAX` (handled by S/M/L2).
```cpp
if (seq_lens[b] <= L_L2_MAX) return;
```

**G. Host pipeline `topk_indexer_forward`:**
1. Allocate `scores_buf`, `indices_buf`, `topk_indices`, `scores_sorted`, `indices_sorted`, `begin_offsets`, `end_offsets`.
2. Launch `emit_short_topk_kernel<<<batch_size, 256>>>`.
3. Launch `compute_scores_kernel<<<dim3(max_num_pages, batch_size), 256>>>`.
4. **One-time dynamic smem set-up** for both block-sort kernels (BlockRadixSort<512,24>::TempStorage ‚âà 512¬∑24¬∑8 + radix histogram ‚âà 98 KB ‚Äî MUST use dynamic smem):
   ```cpp
   static bool smem_set = false;
   if (!smem_set) {
       cudaFuncSetAttribute((const void*)block_topk_kernel<BT_THREADS, BT_ITEMS>,
           cudaFuncAttributeMaxDynamicSharedMemorySize, 98304);
       cudaFuncSetAttribute((const void*)block_topk_kernel_large<L2_THREADS, L2_ITEMS>,
           cudaFuncAttributeMaxDynamicSharedMemorySize, 102400);   // 100 KB, safe H100
       smem_set = true;
   }
   ```
5. Launch `block_topk_kernel<256, 24><<<batch_size, 256>>>`.
6. **Launch `block_topk_kernel_large<512, 24><<<batch_size, 512>>>` (NEW).**
7. Launch `generate_offsets_begin_end` (modified).
8. Two CUB `DeviceSegmentedRadixSort::SortPairsDescending` calls (size query + actual).
9. Launch `extract_topk_kernel<<<batch_size, 1024>>>` (modified).
10. Return `topk_indices`.

### Phase 3: The "Safety Valve" (CRITICAL)

1. **Strict partition invariant:** Each batch `b` falls into exactly one of {S, M, L2, L} via `L = seq_lens[b]`. Each kernel writes `topk_indices[b,:]` only for its tier and early-returns otherwise. No race possible.
2. **Exact top-K in every tier:**
   - S: enumeration (set-correct when `L ‚â§ K`).
   - M: `cub::BlockRadixSort` ‚Äî deterministic exact sort of 6144 items.
   - **L2: `cub::BlockRadixSort<float, 512, 24>` ‚Äî deterministic exact sort of 12288 items.**
   - L: device-wide segmented radix sort (parent's proven path).
3. **Padding sentinel:** `-1e30f` for out-of-range lanes in TIER-L2 loads (more negative than NaN sentinel `-1e20f`, so NaN-corrected scores still outrank padding). Positions `‚â• min(K,L)` after sort receive `-1`.
4. **NaN guard:** preserved in `compute_scores_kernel` (`if (isnan(s)) s = -1e20f`).
5. **Smem budget verification:** `BlockRadixSort<float, 512, 24, int32_t>::TempStorage` ‚âà `24¬∑512¬∑(4+4) + radix_digits¬∑512¬∑4` ‚âà 98 KB + 16 KB ‚âà 114 KB ‚Äî MIGHT be too large. **Mitigation:** if compile / runtime error, drop to `<512, 16>` (8192 items, covers 6144 < L ‚â§ 8192, ~64 KB temp) ‚Äî still strictly better than parent. Use `cudaFuncSetAttribute(..., 102400)` or up to H100 max 227KB (`cudaDevAttrMaxSharedMemoryPerBlockOptin`).
6. **Fallback ladder (if any TIER-L2 failure):** set `L_L2_MAX = L_MED_MAX` in one line ‚Üí TIER-L2 becomes empty, behavior reverts exactly to parent (9.564). Zero regression risk.
7. **Variable-axis support:** no hardcoded batch/page assumptions.
8. **Static assertion:** `static_assert(L2_THREADS * L2_ITEMS >= L_L2_MAX);` enforces buffer sufficiency at compile time.
9. **Ordering in host pipeline:** TIER-M kernel and TIER-L2 kernel both write `topk_indices[b,:]` ‚Äî but because they have **mutually exclusive entry conditions on `L`**, they cannot write the same row. Order between steps 5 and 6 is irrelevant for correctness.
