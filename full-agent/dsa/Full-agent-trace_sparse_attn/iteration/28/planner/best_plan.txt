## Phase 1: Mathematical Definition

### 1.1 Operator (unchanged)
For each token `t ∈ [0,T)` and head `h ∈ [0,16)`:
- `s_j = dot(q_nope[t,h], k_nope(raw_j)) + dot(q_pe[t,h], k_pe(raw_j))` for each valid `raw_j = sparse_indices[t,j] ≠ -1`.
- `z_j = sm_scale * s_j`.
- `out[t,h,:] = Σ_j softmax(z)_j * k_nope(raw_j)`.
- `lse[t,h] = log2(Σ_j exp(z_j))`.

### 1.2 Split-K FP32 partial state (unchanged)
Partition 2048 slots into `S=64` chunks of 32 slots each. For each `(t,h,s)` compute exact FP32 online-softmax partial state `(m_s, l_s, O_s ∈ R^512)`.

### 1.3 Exact merge (unchanged)
- `m_global = max_s m_s`.
- `scale_s = 0 if m_s == -INF else exp(m_s - m_global)`.
- `l_global = Σ_s l_s * scale_s`.
- `O_global = Σ_s O_s * scale_s`.
- `out = O_global / l_global`; `lse = m_global * log2(e) + log2(l_global)`.

### 1.4 Variables X (the only thing changing)
- **Old layout:** `O_tmp[T, S, 16, 512]`, offset `= t*S*16*512 + s*16*512 + h*512 + d`.
- **New layout:** `O_tmp[T, 16, S, 512]`, offset `= t*16*S*512 + h*S*512 + s*512 + d`.
- `m_tmp[T,16,S]` — unchanged from parent (already reducer-native).
- `l_tmp[T,16,S]` — unchanged from parent (already reducer-native).
- `S = 64`, threshold = 128 tokens (unchanged).

### 1.5 Constraints C
1. Output shapes/dtypes identical; ABI identical; PYBIND11_MODULE footer preserved.
2. All accumulation in FP32; BF16 only at input/output boundaries; explicit BF16 intrinsics.
3. `sparse_indices == -1` produces neutral contribution.
4. `lse` in base 2.

### 1.6 Objective f(X)
Minimize reducer memory-read latency under unchanged compute cost. Expected: reducer is currently the bottleneck for small-token path due to poor `O_tmp` access stride; after re-layout, inner `for s` loop reads 64 consecutive `float4`s at stride 2 KB instead of 32 KB, fitting well in L2.

### 2.4 Reducer micro-improvement (bundled, optional but recommended)

Specifically, in the reducer:
- lane-0 computes `m_max`, `l_sum`, and then writes `scale_smem[i] = (m_s_smem[i] != -INF) ? exp(m_s_smem[i] - m_max) : 0` (unnormalized scale).
- After `__syncthreads`, compute `inv_l = 1/l_global` once and use `coeff = scale_smem[s] * inv_l` in the inner loop. (Parent already has this pattern; just keep it.)

## Phase 3: The "Safety Valve" (CRITICAL)

### 3.1 Mathematical equivalence
Both producer and consumer use identical offset formula `t*16*S*512 + h*S*512 + s*512 + d`. This is a pure permutation of a 4D tensor — no data is reinterpreted, no reduction order changed. The exact same FP32 bits are written and read. Therefore correctness is preserved **by equivalence of indexing**, not by approximation.

### 3.2 Neutral-state masking (unchanged from parent)
`sparse_indices == -1` paths skip `O_reg` updates; empty partitions store `m = -INF`, `l = 0`, `O = 0`. Reducer's `m_s_smem[i] != -INF` check and final `(m_max == -INF || l_sum <= 0.0f) ? -INFINITY` branch prevent NaNs.

### 3.3 Alignment safety
- `s * 512` is a multiple of 512 floats = 2048 bytes, well-aligned for `float4` (16B) and `float2` (8B).
- `j * 64 + lane * 2` offset within 512 preserves `float2` alignment (parent invariant).
- `d_idx = lane * 4` preserves `float4` alignment (parent invariant).
- Total `t*16*S*512` base is a multiple of 512 — safe.
No alignment regression possible.

### 3.4 Dispatch fallback
Large-token path (`num_tokens >= 128`) is byte-identical to parent's fused kernel — no possible regression. Only the split-K path (`num_tokens < 128`) is modified.

### 3.5 Build-environment enforcement
- `#include <torch/extension.h>` and `#include <cuda_bf16.h>` at top.
- Only explicit BF16 intrinsics (`__bfloat1622float2`, `__floats2bfloat162_rn`).
- Exact entry-point signature preserved.
- Mandatory `PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) { m.def("dsa_forward", &dsa_forward, "DSA Forward"); }` at end.

### 3.6 Correctness test reasoning
The transformation is a tensor-layout permutation applied symmetrically on one writer and one reader. It is trivially equivalent to the parent at the bitwise level in FP32 workspace. Since parent correctness = 1.0, child correctness = 1.0 guaranteed.
