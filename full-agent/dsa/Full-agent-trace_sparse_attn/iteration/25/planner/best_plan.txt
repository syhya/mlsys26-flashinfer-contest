## Phase 1: Mathematical Definition

Per (token t, head h):
- q = concat(q_nope[t,h], q_pe[t,h]) in R^576.
- For each j in [0,2048): raw=sparse_indices[t,j]. If raw==-1, neutral. Else v=ckv_cache[raw/64, raw%64, :], k_pe=kpe_cache[raw/64, raw%64, :], logit z_j = sm_scale*(<q_nope,v>+<q_pe,k_pe>).
- out[t,h] = Σ w_j v_j, w_j=softmax(z). lse=log2(Σ exp(z_j)).

Online softmax state (m, l, O in R^512). Neutral: m=-INF, l=0, O=0.

Partition 2048 slots into S disjoint chunks of size 2048/S. Each partition p produces (m_p, l_p, O_p). Exact merge:
- m = max_p m_p
- scale_p = exp(m_p - m) if m_p!=-INF else 0
- l = Σ l_p * scale_p
- O = Σ O_p * scale_p
- out = O/l if l>0 else 0
- lse = m*log2(e) + log2(l) if l>0 else -INF

**Variables X:**
- S = 32 (was 64)
- slots_per_partition = 2048/S = 64 (was 32)
- inner loop: each compute block processes 2 tiles of 32 slots sequentially reusing smem.

**Constraints C:** Exact mask for -1; FP32 accumulation; BF16 store; base-2 lse; explicit BF16 intrinsics only; preserve ABI + PYBIND11_MODULE.

**Objective:** Minimize small-T kernel latency.

### Step 3: split_k_reduce_kernel — change partition count from 64 to 32
```cpp

__syncthreads();
if (lane == 0) {
    float m_max = -INFINITY;
    for (int i=0;i<32;++i) m_max = fmaxf(m_max, m_s_smem[i]);
    float l_sum = 0.f;
    for (int i=0;i<32;++i) {
        if (m_s_smem[i] != -INFINITY) {

            scale_smem[i] = scale;
            l_sum = fmaf(l_tmp[t*16*32 + h*32 + i], scale, l_sum);

                     : fmaf(m_max, 1.4426950408889634f, __log2f(l_sum));
}
__syncthreads();
// output accumulation: inner loop over 32 partitions (was 64)
int d_idx = lane * 4;
if (d_idx < 512) {
    float out0=0, out1=0, out2=0, out3=0;
    if (m_global != -INFINITY && l_global > 0.f) {
        float inv_l = __fdividef(1.0f, l_global);
        #pragma unroll 8
        for (int s = 0; s < 32; ++s) {              // was 64
            float coeff = scale_smem[s] * inv_l;
            if (coeff > 0.f) {

                out2 = fmaf(val.z, coeff, out2);

}
```

## Phase 3: Safety Valve

1. **Masking:** Inner loop retains `if (idx_shared[i] == -1) continue;` — invalid slots contribute nothing. Load path retains `if (idx != -1)` guard before cache reads.
2. **Empty partition:** If all 64 slots in partition p are -1, the two sub-tile loops do nothing, so m=-INF, l=0, O=0 is written — neutral state, handled exactly in reducer.
3. **Empty row:** Reducer checks `m_max==-INFINITY || l_sum<=0.f` and writes zero output + -INF lse.
4. **Correctness proof:** Online softmax is associative; merging (m,l,O) over slots [p*64, p*64+64) processed as two consecutive 32-tiles yields identical FP32 state to merging over 64 slots atomically. No new numerical path — just re-grouping.
5. **BF16 intrinsics:** Keep every `__bfloat1622float2`, `__floats2bfloat162_rn`. No implicit casts.
6. **ABI:** dsa_forward signature + final PYBIND11_MODULE block preserved byte-for-byte.
7. **Fallback:** If at integration time any compile issue surfaces, reverting S to 64 is a one-line change.
