## Phase 1: Mathematical Definition

### 1.1 Operator (unchanged)
For token t, head h:
- logit_j = sm_scale * (⟨q_nope[t,h], k_nope(raw_j)⟩ + ⟨q_pe[t,h], k_pe(raw_j)⟩)
- out[t,h] = Σ_valid exp(logit_j)/Z · k_nope(raw_j),  Z = Σ exp(logit_j)
- lse[t,h] = log2(Z) if any valid else -INF
- raw_j == -1 contributes nothing.

### 1.2 Split-K sufficient statistics (unchanged)
Partition 2048 slots into S=64 tiles of 32. Each (t,h,s) computes (m_s, l_s, O_s) in FP32 registers via online softmax.

### 1.3 Merge (unchanged, exact)
m = max_s m_s; coeff_s = exp(m_s-m)/L where L = Σ_s l_s exp(m_s-m); out = Σ_s coeff_s · O_s.

### 1.4 The ONE change: storage precision
O_s is stored in workspace as BF16 (via __floats2bfloat162_rn) instead of FP32. m_tmp, l_tmp remain FP32 (they are scalars, negligible).

Variables X: dtype(O_tmp) ∈ {FP32, BF16} → pick BF16.
Constraints C: numerical error per BF16 store ≤ 2⁻⁷ · |value|. Since each O_s entry is bounded by l_s (max exp-sum over 32 slots), and final normalization divides by L ≥ max_s l_s · exp(m_s-m), the final error stays within bf16 rtol=1e-2 atol=1e-2.
Objective f(X): minimize reducer HBM read bytes. FP32→BF16 halves the reducer's dominant read.

    __nv_bfloat162 packed = __floats2bfloat162_rn(O_reg[j].x, O_reg[j].y);

### 2.3 Modified `split_k_reduce_kernel`
Adopt the sibling 94736692 simplification (remove `l_s_smem[64]`; compute coeff directly using `l_tmp` inside lane-0 block) AND change O_tmp read dtype:
- Signature: `const __nv_bfloat16* O_tmp`.
- Inside the 64-partition output accumulation loop, each thread handles 4 consecutive output dims (d_idx = lane*4). Load 4 BF16 values as one `uint64_t` (or two `__nv_bfloat162`) from `&O_tmp[base_idx]`, convert to 2 float2 via `__bfloat1622float2`, then FMA with `coeff`:
```
uint2 raw = *reinterpret_cast<const uint2*>(&O_tmp[base_idx]);
__nv_bfloat162 lo = *reinterpret_cast<__nv_bfloat162*>(&raw.x);
__nv_bfloat162 hi = *reinterpret_cast<__nv_bfloat162*>(&raw.y);
float2 v01 = __bfloat1622float2(lo);
float2 v23 = __bfloat1622float2(hi);
out0 = fmaf(v01.x, coeff, out0);
out1 = fmaf(v01.y, coeff, out1);
out2 = fmaf(v23.x, coeff, out2);
out3 = fmaf(v23.y, coeff, out3);
```
- Accumulation remains FP32. Final BF16 pack store unchanged.
- Inside the lane-0 scalar setup block, compute `coeff_s = exp(m_s - m_global) / l_global` directly from `l_tmp[...]` (sibling 94736692 pattern).

## Phase 3: The "Safety Valve" (CRITICAL)

### 3.1 Numerical bound guarantee
Per-slot contribution to O_s is `beta_j · v_j` with `|beta_j| ≤ 1` after rescaling and `|v_j|` bounded by input BF16 range (≤ ~3.4e38 theoretically, realistic ≤ 1e2). After 32 slots per partition, `|O_s| ≤ 32 · max|v|`. Final store rounds to BF16 with relative error ≤ 2⁻⁷ ≈ 0.78%, well under the 1e-2 tolerance even after 64-partition accumulation (errors are incoherent, scaled by coeff_s which sums to 1).

### 3.2 Neutral-state masking (unchanged)
raw==-1 → skipped before cache load. Partitions with all-invalid slots store m=-INF, l=0, O=0 (BF16 zero = exact zero, safe under merge).

### 3.3 Empty-row fallback (unchanged)
Reducer detects `m_global == -INF || l_global <= 0` → writes zero output and -INF lse.

### 3.4 Deterministic fallback
If any compile issue with BF16 workspace reinterpret arises, the executor must revert `O_tmp` dtype to kFloat32 and restore FP32 reads in reducer. This reproduces sibling 94736692 exactly → guaranteed score 12.63.

### 3.5 ABI/build safety (unchanged)
- `#include <torch/extension.h>`, `#include <cuda_bf16.h>`, `#include <tuple>`.
- Explicit BF16 intrinsics only: `__bfloat1622float2`, `__floats2bfloat162_rn`.
- Mandatory `PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) { m.def("dsa_forward", ...); }` footer.
- Exact C++ signature unchanged.

if (num_tokens < 128) {
    constexpr int S = 64;
    auto O_tmp = torch::empty({num_tokens, S, 16, 512},
                              torch::dtype(torch::kBFloat16).device(q_nope.device()));
    auto m_tmp = torch::empty({num_tokens, 16, S}, torch::dtype(torch::kFloat32).device(q_nope.device()));
    auto l_tmp = torch::empty({num_tokens, 16, S}, torch::dtype(torch::kFloat32).device(q_nope.device()));

    split_k_compute_kernel<<<dim3(S,num_tokens,1), dim3(32,16,1)>>>(
        reinterpret_cast<const __nv_bfloat16*>(q_nope.data_ptr<at::BFloat16>()),
        reinterpret_cast<const __nv_bfloat16*>(q_pe.data_ptr<at::BFloat16>()),
        reinterpret_cast<const __nv_bfloat16*>(ckv_cache.data_ptr<at::BFloat16>()),

        sparse_indices.data_ptr<int32_t>(),
        sm_scale,
        reinterpret_cast<__nv_bfloat16*>(O_tmp.data_ptr<at::BFloat16>()),

    split_k_reduce_kernel<<<dim3(num_tokens,16,1), dim3(128,1,1)>>>(
        reinterpret_cast<const __nv_bfloat16*>(O_tmp.data_ptr<at::BFloat16>()),
        m_tmp.data_ptr<float>(), l_tmp.data_ptr<float>(),
        reinterpret_cast<__nv_bfloat16*>(output.data_ptr<at::BFloat16>()),
        lse.data_ptr<float>());
} else {
    dsa_forward_kernel<<<dim3(num_tokens), dim3(32,16)>>>(... unchanged ...);
}
return {output, lse};
