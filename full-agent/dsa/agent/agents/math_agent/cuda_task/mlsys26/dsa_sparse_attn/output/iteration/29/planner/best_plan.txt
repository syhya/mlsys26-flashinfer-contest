## Phase 1: Mathematical Definition

### 1.1 Operator (unchanged from reference)
For each `(t, h)`:

- `lse[t,h] = m * log2(e) + log2(Σ exp(s_j - m))`, else `-INF`

This is byte-equivalent to the best sibling's math.

### 2.1 Branch base

### 2.2 Optimization 1: Reducer-native head-major workspace layout
Change `O_tmp` from `[T, S=64, 16, 512]` to `[T, 16, S=64, 512]`.

**Producer side** (`split_k_compute_kernel`):
- For block `(s, t)`, head `h`:
  - `base_idx = ((t * 16 + h) * S + s) * 512 + j*64 + lane*2`
- This is exactly one `float2` per lane per `j`, fully coalesced within a warp because lanes vary in the innermost dim.

**Consumer side** (`split_k_reduce_kernel`):
- For block `(t, h)`, lane handles `d_idx = lane*4`:
  - For `s ∈ [0, 64)`: `base_idx = ((t * 16 + h) * S + s) * 512 + d_idx`
- The 64 reads per lane are now to addresses spaced exactly 512 floats apart within one contiguous `[64, 512]` slab — much more L2-friendly than strided across heads.

This is purely an indexing change. Math is byte-identical.

### 2.3 Optimization 2: Partition-level all-invalid early skip
In `split_k_compute_kernel`, after loading `idx_shared[32]`:
```
int valid_mask = __ballot_sync(0xffffffff, (tid < 32) ? (idx_shared[tid] != -1) : 0);
if (valid_mask == 0) {
    // write neutral state and exit
    if (lane == 0) {
        m_tmp[((t*16)+h)*S + s] = -INFINITY;
        l_tmp[((t*16)+h)*S + s] = 0.0f;
    }
    // O_tmp values are unused when l_p == 0 in reducer (coeff_p = 0); but to be safe,
    // we can skip writing O_tmp because the reducer must already gate on m_p == -INFINITY.
    return;
}
```
**Correctness gate:** the reducer must explicitly check `m_p == -INFINITY` and set `coeff_p = 0` (the best sibling already does this — confirmed). So skipping `O_tmp` writes is safe.

Also re-layout `m_tmp` and `l_tmp` to `[T, 16, S]` indexing (already that shape in best sibling).

### 2.4 Optimization 3: Reducer simplification preserved
Keep the best sibling's reducer pattern: lane-0 computes `m_global`, `l_global`, and writes `lse`; precomputes `scale_smem[s] = exp(m_s - m_global)` (skip `-INF`); then each lane multiplies-accumulates 64 partials. **Use `inv_l = 1/l_global` and fold into `coeff` once, exactly like the best sibling.**

## Phase 3: The "Safety Valve" (CRITICAL)

### 3.1 Neutral-state masking
- Producer: when `valid_mask == 0`, write `m_tmp = -INF`, `l_tmp = 0`, do not touch `O_tmp` for that (t,h,s) slab. Reducer must (and does) treat `m_p == -INF` as `coeff_p = 0`, so unread `O_tmp` is never multiplied.
- Per-key `idx == -1`: skip the inner-loop iteration (already done).

### 3.2 Reducer correctness gates (preserve from best sibling)
- `m_global = max_s m_tmp[t,h,s]`. If all `-INF`, set `out=0`, `lse=-INF`.
- `l_global = Σ_s l_tmp * exp(m_s - m_global)` skipping `-INF` entries.
- If `l_global <= 0`, force zero output and `lse = -INF`.
- Final `out = Σ_s coeff_s * O_tmp[t,h,s,:]`, where `coeff_s = exp(m_s-m_global)/l_global` (0 if `m_s == -INF`).

### 3.3 Layout consistency check (mandatory in implementation)
Producer index formula and consumer index formula MUST be derived from the same closed-form expression `((t*16+h)*S + s)*512 + d`. Write a single device-side `__forceinline__` helper:
```
__device__ __forceinline__ int otmp_idx(int t, int h, int s, int d, int S) {
    return ((t * 16 + h) * S + s) * 512 + d;
}
```
Use it in both kernels. This eliminates the most common bug class.

### 3.4 Conservative fallback
- Keep large-path fused kernel byte-identical to best sibling (it works at correctness 1.0).
- If executor is uncertain about layout change, the fallback is to keep the original layout `[T,S,16,512]` from the best sibling and ship only the early-skip optimization. This still produces a strictly better-or-equal child.

### 4.4 Kernel 2: `split_k_compute_kernel` (modified small path)
Block: `(32, 16)`, Grid: `(S=64, T)`.
1. Load `idx_shared[32]` from `sparse_indices[t*2048 + s*32 + tid]` for `tid < 32`.
2. `__syncthreads();`
3. **Early-skip (NEW):** lane 0 of warp 0 computes:
   ```cpp
   __shared__ int partition_valid;
   if (tid == 0) {
       int any = 0;
       #pragma unroll
       for (int i = 0; i < 32; ++i) any |= (idx_shared[i] != -1);
       partition_valid = any;
   }
   __syncthreads();
   if (!partition_valid) {
       if (tid == 0) {
           // write neutral state for all 16 heads
           #pragma unroll
           for (int hh = 0; hh < 16; ++hh) {
               m_tmp[ml_idx(t, hh, s, 64)] = -INFINITY;
               l_tmp[ml_idx(t, hh, s, 64)] = 0.0f;
           }
       }
       return;
   }
   ```
4. Stage `smem_Kc` and `smem_Kp` (same `float4` cooperative load as best sibling, gated by `idx != -1`).
5. Inner loop over 32 keys: compute logit, online-softmax update (FP32). Identical to best sibling.
6. Write outputs using **NEW layout**:
   ```cpp

   for (int j = 0; j < 8; ++j) {
   ```

### 4.5 Kernel 3: `split_k_reduce_kernel` (modified small path)
Block: `128`, Grid: `(T, 16)`.
1. Load `m_s_smem[64]` from `m_tmp[ml_idx(t,h,s,64)]` for `lane < 64`.
2. `__syncthreads()`.
3. Lane 0:
   - Compute `m_max = max_s m_s_smem[s]`.
   - Compute `l_sum = Σ_s l_tmp[ml_idx(t,h,s,64)] * exp(m_s - m_max)` for `m_s != -INF`; store `scale_smem[s] = exp(m_s - m_max)` (or 0 if -INF).
   - `m_global = m_max; l_global = l_sum;`
   - `lse[t*16+h] = (m_max == -INF || l_sum <= 0) ? -INF : fmaf(m_max, 1.4426950408889634f, __log2f(l_sum));`
4. `__syncthreads()`.
5. `inv_l = (l_global > 0) ? 1.0f / l_global : 0.0f;`
6. `d_idx = lane * 4` (lanes 0..127 cover 512). For each lane:
   ```cpp
   float4 acc = make_float4(0,0,0,0);
   if (m_global != -INFINITY && l_global > 0.0f) {
       #pragma unroll 8
       for (int s = 0; s < 64; ++s) {
           float coeff = scale_smem[s] * inv_l;
           if (coeff > 0.0f) {
               int idx = otmp_idx(t, h, s, d_idx, 64);
               float4 v = *reinterpret_cast<const float4*>(&O_tmp[idx]);
               acc.x = fmaf(v.x, coeff, acc.x);
               acc.y = fmaf(v.y, coeff, acc.y);
               acc.z = fmaf(v.z, coeff, acc.z);
               acc.w = fmaf(v.w, coeff, acc.w);
           }
       }
   }
   __nv_bfloat162 b0 = __floats2bfloat162_rn(acc.x, acc.y);
   __nv_bfloat162 b1 = __floats2bfloat162_rn(acc.z, acc.w);
   uint2 out = make_uint2(*reinterpret_cast<uint32_t*>(&b0), *reinterpret_cast<uint32_t*>(&b1));
   *reinterpret_cast<uint2*>(&output_ptr[(t*16+h)*512 + d_idx]) = out;
   ```
   **Note the contiguous stride pattern:** all 64 reads per lane are now within the same `[64, 512]` head-local slab → much better L2 hit-rate than the original slot-major layout.
