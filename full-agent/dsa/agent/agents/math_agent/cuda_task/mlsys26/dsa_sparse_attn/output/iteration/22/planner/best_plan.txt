## Phase 1: Mathematical Definition

### 1.1 Operator
For each token `t in [0,T)` and head `h in [0,16)`:
- `q_nope[t,h] in R^512`, `q_pe[t,h] in R^64`
- For sparse slot `j in [0,2048)`: `raw = sparse_indices[t,j]`; if `raw==-1`, contributes nothing; else `k_nope = ckv_cache[raw/64, raw%64, :]`, `k_pe = kpe_cache[raw/64, raw%64, :]`
- `z_j = sm_scale * (dot(q_nope, k_nope) + dot(q_pe, k_pe))`
- `out[t,h] = sum_valid softmax(z)_j * k_nope_j`
- `lse[t,h] = log2(sum_valid exp(z_j))`

### 1.2 Online softmax sufficient statistics (m, l, O)
- `m_new = max(m, z)`, `alpha = exp(m-m_new)` (0 if m=-INF), `beta = exp(z-m_new)`
- `l_new = l*alpha + beta`, `O_new = O*alpha + beta*v`

### 1.3 Exact split-K merge
For partials A, B: `m=max(m_a,m_b)`, `s_a=exp(m_a-m)`, `s_b=exp(m_b-m)`, `l = l_a*s_a + l_b*s_b`, `O = O_a*s_a + O_b*s_b`.

### 1.4 Variables, Constraints, Objective
- **X:** dispatch threshold `tau=128`, split partitions `S=64`, reducer block 128, fast-math intrinsics.
- **C:** FP32 accum; BF16 only at final store; `-1` masked; base-2 lse; explicit BF16 intrinsics; correct ABI.
- **f(X):** minimize latency subject to correctness=1.0.

## Phase 3: The Safety Valve (CRITICAL)

### 3.1 Masking
- Skip any sparse slot with `raw == -1` BEFORE any cache load and before any score update.
- Empty partitions naturally produce `m=-INF, l=0, O=0` which the merge formula treats as identity.

### 3.2 Empty-row repair (in reducer)
```
if (m_max == -INFINITY || l_sum <= 0.0f) {
    output[t,h,:] = 0;
    lse[t,h] = -INFINITY;
}
```
Same in the fused path: `inv_l = (l > 0) ? 1/l : 0`; lse = `(l > 0) ? m*log2e + log2(l) : -INF`.

### 3.3 BF16 intrinsics enforcement
- BF16->float: `__bfloat162float`, `__bfloat1622float2`.
- float->BF16: `__float2bfloat16`, `__floats2bfloat162_rn`.
- Required because build uses `-D__CUDA_NO_BFLOAT16_CONVERSIONS__`.

### 3.4 Correctness fallback guarantee
Because we copy a verified score=12.63 implementation byte-for-byte (modulo the trivial intrinsic substitutions which are no-ops semantically), correctness is mathematically inherited.

    #pragma unroll
    for (int offset = 16; offset > 0; offset /= 2)

### 4.3 Three kernels (copy from ancestor 94736692)

#### Kernel A: `dsa_forward_kernel` (large path)
- Grid: `(num_tokens)`, Block: `(32, 16)`.
- Shared: `idx_shared[32]`, `tile_has_valid` (int), `smem_Kc[32*512]`, `smem_Kp[32*64]` BF16.
- Each thread: lane = `threadIdx.x`, h = `threadIdx.y`.
- Hoist `q_nope[t,h,:]` and `q_pe[t,h,:]` to FP32 registers (8 float2 + 1 float2).
- Outer loop over 64 tiles: load 32 indices, set `tile_has_valid=0`, scan, skip if all -1.
- Stage Kc (4 steps × 8 rows × 64 float4 cols) and Kp (256 threads × 8 cols).
- Inner loop over 32 keys: skip -1, compute warp dot, broadcast via `__shfl_sync(.., 0)`, multiply by sm_scale, online-softmax update (`__expf`), accumulate O.
- Final: `inv_l = (l > 0) ? __fdividef(1, l) : 0`, store BF16, lane 0 writes `lse = (l > 0) ? fmaf(m, 1.4426950408889634f, __log2f(l)) : -INFINITY`.

#### Kernel B: `split_k_compute_kernel` (small path, partial)
- Grid: `(S=64, num_tokens)`, Block: `(32, 16)`.
- Same body as Kernel A but processes ONE tile `s = blockIdx.x` only.
- Stores `O_tmp[t, s, h, :]` as 8×float2 per lane, `m_tmp[t,h,s]`, `l_tmp[t,h,s]`.
- DOES NOT normalize.

#### Kernel C: `split_k_reduce_kernel` (small path, reducer)
- Grid: `(num_tokens, 16)`, Block: `128`.
- Lane<64 loads `m_s_smem[lane] = m_tmp[t,h,lane]`.
- Lane 0 computes `m_max = max over 64`, then `l_sum = sum_i l_tmp[t,h,i] * exp(m_s_smem[i] - m_max)` (skip -INF), writes `lse_ptr[t,h] = (l_sum>0) ? fmaf(m_max, 1.4426950408889634f, __log2f(l_sum)) : -INFINITY`.
- Each thread handles 4 output dims (`d = lane*4`); for each partition s with valid m, compute `coeff = exp(m_s_smem[s] - m_global) * inv_l`, accumulate `float4 val = O_tmp[t,s,h,d..d+3] * coeff`. Pack to BF16 pair, write as `uint2`.
