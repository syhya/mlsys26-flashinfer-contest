By fetching 8 tokens worth of K-cache into registers before starting the loop over 64 heads, the kernel drastically reduces redundant memory access instructions and maximizes compute-to-memory-access ratio. *   **Key Insight:** For bandwidth-bound dot products, maximizing register reuse and instruction-level parallelism through extreme loop unrolling of small, deterministic loops (like the 8-token chunk) yields massive performance dividends, even without complex Tensor Core abstractions.
*   **Identified Risk:** Register pressure is a significant risk. Declaring arrays of `float4` (like `k_f[8]`) and large unrolled variable arrays (`val[8]`) per thread consumes a vast amount of registers. This could lead to register spilling to local memory if the architecture limits are hit, which would catastrophically degrade performance.
